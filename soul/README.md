# soul/

Foundational documents that define what Loam is and why it exists. These are stable — they change when beliefs change, not when implementations change.

| Document | Purpose |
|---|---|
| [PHILOSOPHY.md](PHILOSOPHY.md) | Why Loam exists and what it believes |
| [IDENTITY.md](IDENTITY.md) | How the Operator and Agent relate in practice |
| [VOCABULARY.md](VOCABULARY.md) | Canonical terms and the four-test naming standard |
| [CONTEXT-STACK.md](CONTEXT-STACK.md) | Authority order for all work |

Everything in `soul/` sits above the harness and the ops layer. An agent reads this directory to understand what it's working on before it reads anything about how.
